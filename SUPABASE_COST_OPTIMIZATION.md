# 🛡️ Supabase 무료 요금제 비용 최적화 및 성능 유지 가이드

이 가이드는 Supabase 무료 티어(Free Tier)의 제한 사항 내에서 서비스 로딩 속도와 데이터베이스 상태를 최상으로 유지하면서 비용을 최적화하는 전략을 정리한 문서입니다.

---

## 1. 데이터베이스(Database) 최적화

### ✅ 불필요한 컬럼 조회 지양 (Select optimization)
`select('*')`는 모든 데이터를 네트워크로 전송하므로 대역폭(Bandwidth) 소모가 큽니다. 필요한 필드만 명시하여 데이터 전송량을 줄이세요.
- **AS-IS**: `.select('*')`
- **TO-BE**: `.select('id, name, price, img_urls')`

### ✅ 인덱스(Index) 활용
조회 성능을 유지하면서 CPU 부하를 줄이려면 필터링이 자주 발생하는 컬럼(`category`, `created_at` 등)에 인덱스를 생성해야 합니다.
- **SQL 예시**: `CREATE INDEX idx_products_category ON products(sub_id);`

### ✅ 효율적인 방문자 통계 업데이트
현재 메인 페이지 로딩 시마다 `increment_visitor_count` RPC를 호출하고 있습니다. 이는 DB 쓰기 작업을 유발합니다.
- **최적화**: 세션당 1회만 호출하도록 클라이언트 측(`localStorage` 또는 `sessionStorage`)에서 제어하거나, Redis(Upstash 등)를 결합하여 쓰기 빈도를 낮추는 것을 고려할 수 있습니다.

---

## 2. 대역폭(Bandwidth/Egress) 및 로딩 속도 최적화

### ✅ Next.js 데이터 캐싱 활용
데이터가 실시간으로 변할 필요가 없다면 Next.js의 캐싱 기능을 최대한 활용하여 Supabase로의 요청 횟수를 줄입니다.
- **ISR 적용**: `export const revalidate = 3600;` (1시간마다 갱신)

### ✅ 페이지네이션(Pagination) 필수 적용
상품 리스트를 한꺼번에 불러오지 말고 `range(from, to)`를 사용하여 필요한 만큼만 끊어서 불러오세요. 대역폭 사용량을 획기적으로 줄일 수 있습니다.

---

## 3. 스토리지(Storage) 및 이미지 최적화

### ✅ Supabase Image Transformation 활용
원본 이미지를 그대로 불러오는 대신, Supabase Storage에서 제공하는 변환 옵션을 사용하여 브라우저에 최적화된 크기로 전달합니다.
- **URL 구성**: `.../storage/v1/object/public/bucket/image.jpg?width=400&quality=75&format=webp`
- **효과**: 이미지 파일 크기가 70~80% 감소하며, 이는 곧 대역폭(Egress) 절약과 로딩 속도 향상으로 이어집니다.

### ✅ 업로드 전 이미지 압축
관리자 페이지에서 상품 이미지를 업로드할 때, 클라이언트 측에서 `browser-image-compression` 같은 라이브러리를 사용하여 미리 압축 후 업로드하면 스토리지 용량을 절약할 수 있습니다.

---

## 4. 무료 티어 유지 및 관리 (Maintenance)

### ✅ 프로젝트 일시정지(Pause) 방지
Supabase 무료 티어는 **1주일간 데이터베이스 활동이 없으면 프로젝트가 자동으로 일시정지**됩니다. 
- **대책**: 최소 하루에 한 번은 관리자 페이지에 접속하거나, 자동화된 스케줄러(GitHub Actions 등)를 통해 간단한 조회를 수행하는 것이 안전합니다.

### ✅ 로그 및 임시 데이터 정리
`auth`와 같은 시스템 로그나 임시 테이블이 차지하는 용량을 주기적으로 확인하세요. 500MB 제한을 지키기 위해 사용하지 않는 대용량 로그는 비워주는 것이 좋습니다.

---

## 5. 핵심 요약 (Cheat Sheet)

| 항목             | 최적화 방법                               | 기대 효과                   |
| :--------------- | :---------------------------------------- | :-------------------------- |
| **조회(Select)** | 필요한 컬럼만 추출 (`select('id, name')`) | 대역폭 절약                 |
| **이미지**       | URL 파라미터로 리사이징 (`?width=400`)    | 로딩 속도 증가, Egress 절약 |
| **캐싱**         | Next.js ISR/Fetch 캐시 적용               | DB 부하 감소, API 호출 절약 |
| **페이지네이션** | `range()` 기반 데이터 로딩                | 초기 로딩 성능 개선         |
| **활동 유지**    | 7일 이내 최소 1회 API 호출                | 서비스 중단 방지            |

이 가이드를 준수하면 Supabase 무료 용량 내에서도 충분히 쾌적하고 안정적인 서비스를 운영할 수 있습니다. 추가로 자동화나 특정 코드 구현이 필요하시면 말씀해 주세요!
